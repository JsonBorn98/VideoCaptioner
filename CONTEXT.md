# Subtitle Postprocessing

字幕后处理负责在字幕内容生成后，评估并改善观众实际看到的字幕质量。

## Language

**初版字幕**:
字幕优化翻译完成后形成、可以直接交付使用的完整字幕；字幕后处理以它作为输入，但不回写其内容。
_Avoid_: 中间字幕、待处理字幕、翻译缓存

**字幕后处理模块**:
位于字幕优化翻译之后、也可被独立调用的可选功能模块，接收完整成型字幕并生成单独的后处理结果。
_Avoid_: 字幕优化翻译、翻译收尾步骤、速度设置页

**功能页面边界**:
每个功能页面只呈现和操作自身模块的输入、设置与结果，不展示其他模块的快捷开关、摘要或设置入口。
_Avoid_: 跨页快捷区、聚合设置页、流程控制页

**字幕后处理页面**:
主页 workflow 中位于字幕优化翻译之后、字幕视频合成之前的功能页面；既显示编排任务，也接受用户手动创建独立后处理任务。
_Avoid_: 字幕速度页、后处理设置页、翻译快捷区

**流程编排层**:
负责为一次完整或批量任务选择要经过哪些功能阶段并确定阶段顺序；任务创建页和批量任务配置属于流程编排层。
_Avoid_: 功能页面、字幕后处理设置、任务线程

**后处理字幕**:
字幕后处理模块根据初版字幕或外部成型字幕生成的新版本，和作为输入的字幕相互独立。
_Avoid_: 覆盖字幕、初版字幕、翻译结果

**活动字幕输出**:
完整 workflow 交给视频合成等下游阶段的字幕；后处理成功时是后处理字幕，关闭或失败时是初版字幕。
_Avoid_: 最终字幕、默认字幕、当前文件

**强介入后处理**:
在用户启用相应能力后，可以大幅调整字幕段结构、时间轴、内容归属和必要文字表达，以使整体显示与阅读节奏满足目标。
_Avoid_: 微调、轻量清理、无损格式化

**后处理能力项**:
字幕后处理模块中可由用户独立启用或关闭的一类调整能力；关闭某项只限制该类调整，不代表跳过整个后处理模块。
_Avoid_: 全局开关、速度预设、处理步骤

**默认后处理组合**:
完整翻译 workflow 自动采用的后处理能力集合：均衡速度优化、必要语义修复和行尾弱标点清理默认开启，其余内容清理、独立间隙修复和媒体对齐默认关闭。
_Avoid_: 全部开启、出厂速度预设、强制处理

**后处理模板**:
系统提供的宽松、均衡和平滑优先三套完整后处理配置；用户可修改其工作值，也可恢复到各自不可变的出厂基线。
_Avoid_: 速度预设、只读内置方案、默认配置

**后处理配置方案**:
一次字幕后处理所需全部能力项和参数的可选择配置，可以是后处理模板，也可以是从模板复制创建的自定义方案。
_Avoid_: 速度配置方案、临时覆盖、设置页面

**当前后处理方案**:
字幕后处理页面当前选中的配置方案；页面对能力项或参数的修改立即持久化到该方案。
_Avoid_: 本次任务覆盖、未保存草稿、运行时快照

**出厂基线**:
随软件版本定义、不可被用户修改的后处理模板原始值，是模板和其派生方案执行恢复默认时的权威来源。
_Avoid_: 当前模板值、创建快照、硬编码默认

**来源模板**:
自定义后处理配置方案复制自的宽松、均衡或平滑优先模板；恢复默认始终使用该模板的出厂基线。
_Avoid_: 父配置、动态继承、当前模板

**内部质量验收**:
后处理提交修改前始终执行的安全与质量判断，不因用户关闭额外报告文件而停用。
_Avoid_: QA 报告文件、可选审计、调试日志

**跳过后处理**:
不调用字幕后处理模块，直接使用字幕优化翻译产生的初版字幕。
_Avoid_: 关闭速度优化、分析模式、处理失败

**独立后处理任务**:
不经过字幕优化翻译、直接将外部完整成型字幕交给字幕后处理模块的任务。
_Avoid_: 重新翻译、字幕优化翻译任务

**编排后处理任务**:
完整 workflow 进入字幕后处理阶段后自动运行、完成后自动续接下一阶段的后处理任务，使用任务开始时冻结的配置。
_Avoid_: 独立后处理任务、后台隐式收尾、翻译后回调

**成型字幕**:
已经具备完整时间轴和可显示文本、可以独立保存或播放的单语或双语字幕。
_Avoid_: 词级时间证据、翻译中间结果、文本稿

**模块交付字幕**:
字幕内容模块共同理解的规范字幕状态，只承载时间轴、显示文本、单双语角色和行序，不承载字体、颜色、定位或特效；连续 workflow 可直接传递该状态，持久化边界统一序列化为 SRT。
_Avoid_: 样式字幕、源字幕副本、文件路径交接

**字幕导入规范化**:
将外部 SRT、VTT 或 ASS 提取为模块可处理的字幕内容；输入文件的表现样式在此边界被有意丢弃。
_Avoid_: 格式无损导入、ASS 往返、样式继承

**普通文本字幕**:
以连续对白文字和时间轴为主要内容、不依赖多轨特效、定位、绘图或卡拉 OK 语义才能正确理解的字幕，是软件内容处理模块承诺支持的输入范围。
_Avoid_: 特效字幕工程、ASS 项目、字幕内容轨道

**样式化导出**:
用户需要特定呈现格式时，由字幕样式模块将模块交付字幕转换为应用当前布局和样式配置对应的成品，主要输出 ASS。
_Avoid_: 模块交付、格式另存为、保留输入样式

**后处理结果工作稿**:
字幕后处理页面中承载最新处理结果并允许人工微调的可编辑字幕内容；后处理完成时自动交付一次 SRT，之后的人工修改须由用户显式保存或导出。
_Avoid_: 输出文件路径、自动同步文件、只读预览

**保存字幕工作稿**:
将内容处理页面当前可编辑结果提交到该任务的规范 SRT 主文件，不选择表现格式，也不改变源输入。
_Avoid_: 导出字幕、另存为、自动同步

**导出字幕成品**:
从当前字幕工作稿另行生成用户指定格式的交付副本；导出不改变规范 SRT 主文件或模块间活动字幕路径。
_Avoid_: 保存字幕、模块交付、切换主文件

**阶段自动保存**:
每个产生字幕内容的 workflow 阶段完成时必须将当时的规范字幕状态写入独立 SRT 成果文件，是不可关闭的任务保障；下游正常执行不需要重新读取该文件，软件重启后也不据此自动续跑。
_Avoid_: 自动导出、缓存、断点恢复、覆盖输入

**阶段自动导出**:
由 workflow 编排层按任务开始前冻结的统一导出策略，在每个实际完成的字幕阶段后从规范工作稿额外生成指定格式和样式的派生成品；它只有一个任务级开关，不替代阶段自动保存。
_Avoid_: 模块输出、线程保存、阶段自动保存

**字幕阶段前缀**:
标识字幕内容所处 workflow 阶段的固定文件名前缀：`【转录字幕】`、`【初版字幕】`、`【后处理字幕】`；同阶段所有保存和导出格式共享该前缀。
_Avoid_: 导出前缀、格式前缀、嵌套阶段前缀

**字幕结构识别**:
确定成型字幕是单语还是双语，并在双语情况下确定原文侧与译文侧顺序的输入解释过程。
_Avoid_: 语言检测、字幕布局渲染、主显示侧选择

**结构识别降级**:
外部字幕无法可靠判定单双语角色时，形成可检查预览并要求独立 GUI 用户确认结构后才能处理；非交互调用则明确警告并采用保守的默认角色解释。
_Avoid_: 自动纠错、静默猜测、解析失败

**字幕结构确认**:
独立 GUI 对低置信度字幕要求用户明确选择单语、原文在上或译文在上后，才允许进入强介入后处理。
_Avoid_: 普通警告、语言选择、字幕布局导出

**后处理降级**:
字幕后处理无法产出合法结果时，完整 workflow 改用未被修改的初版字幕继续执行，并明确报告后处理失败。
_Avoid_: 静默跳过、任务失败、局部回滚

**字幕段**:
一段具有明确开始和结束时间的字幕内容，是字幕阅读速度评估的基本时间单元。
_Avoid_: 字幕块、字幕行、segment

**显示侧**:
一个字幕段中实际呈现给观众的一路文本，可以是原文侧或译文侧；双语布局同时包含两个显示侧。
_Avoid_: 字段、text、translated_text

**双语侧策略**:
双语字幕默认以译文作为主显示侧，原文作为参考显示侧；结构和时间修改保持两侧同步，文本修改默认只作用于译文。
_Avoid_: 上下行策略、布局顺序、只处理译文

**同时优化两侧**:
用户显式允许字幕后处理分别优化原文与译文文本的双语侧策略，仍须保持两侧结构对应和语义完整。
_Avoid_: 双语模式、参考侧审计、同步拆分

**阅读速度**:
一个显示侧在其显示时长内带给观众的文字阅读负荷，通常用每秒字符数（CPS）表达。
_Avoid_: 播放速度、语速

**硬超速**:
阅读速度超过可接受上限的显示侧，预计大多数观众难以在当前时长内完成阅读。
_Avoid_: 硬警告、严重告警

**舒适超速**:
阅读速度尚未达到硬超速，但已经高于舒适阅读目标的显示侧。
_Avoid_: 舒适警告、软超速

**结构级速度优化**:
一种可以调整字幕段边界、内容归属和文字表达的阅读速度优化能力，关注连续观看过程而不只处理单个超速段。
_Avoid_: C 层处理、强制重切字幕

**速度策略**:
一组共同定义阅读速度目标、安全限制和允许调整范围的字幕后处理规则，可以由用户在预设基础上覆盖高级参数。
_Avoid_: 参数包、算法配置

**速度预设**:
面向常见观看需求预先定义的速度策略，为不需要逐项调参的用户提供稳定起点。
_Avoid_: 固定模式、硬编码方案

**自适应速度目标**:
一种随邻近字幕的内容节奏平缓变化、同时受全局舒适范围和硬上限约束的阅读速度目标。
_Avoid_: 固定 CPS、全片统一速度

**速度跳变**:
连续观看时，相邻显示侧之间出现的明显阅读速度变化。
_Avoid_: 速度方差、单段超速

**时间证据**:
用于判断字幕内容可以安全开始、结束或重新分配的时序信息；不同来源具有不同可信度。
_Avoid_: 时间戳来源、边界数据

**节奏重置点**:
结束上一段连续阅读节奏、使后续字幕可以建立新局部速度目标的明显停顿。
_Avoid_: 普通字幕间隙、切段点

**严格硬限**:
一种优先消除硬超速的可选速度策略，允许在明确限制和可追踪记录下采用更强的内容压缩。
_Avoid_: 强制合规、无损压缩

**未解决超速**:
因语义完整或可信时间边界具有更高优先级而被保留、需要进入质量复查的硬超速。
_Avoid_: 压缩失败、忽略告警

**主显示侧**:
阅读体验驱动速度优化目标的显示侧；翻译任务默认以译文作为主显示侧。
_Avoid_: 主字段、上方字幕

**参考显示侧**:
帮助观众在主显示侧难以理解时核对内容的辅助显示侧，默认只接受审计而不驱动速度优化。
_Avoid_: 次要字段、无关字幕

**单向平滑**:
只降低局部阅读速度尖峰、不主动缩短较慢字幕显示时间的速度优化方式。
_Avoid_: 只修超速、保守模式

**双向平滑**:
在降低快速字幕速度的同时，也允许受限地缩短明显偏慢字幕，使局部阅读节奏向自适应目标收敛。
_Avoid_: 强制统一速度、整体加速

**受保护字幕段**:
因内容用途、显示意图或识别不确定性而不应被自动缩短或结构重排的字幕段。
_Avoid_: 例外字幕、跳过项

**显式保护规则**:
由用户明确指定、优先于自动判断的受保护字幕段识别条件。
_Avoid_: 排除列表、特殊补丁

**确定性优化阶段**:
不依赖生成式模型、对相同输入和速度策略产生可重复结果的速度优化阶段。
_Avoid_: 规则修补、本地模式

**语义修复阶段**:
仅在确定性优化仍无法满足速度目标时，对局部字幕内容进行受验证改写的后续阶段。
_Avoid_: 全量重译、LLM 压缩开关

**语义窗口**:
语义修复阶段能够共同理解和调整的一组连续字幕段，其边界不能跨越受保护字幕段或节奏重置点。
_Avoid_: LLM 批次、整段上下文

**局部回滚**:
语义窗口未通过验证时，仅撤销该窗口产生的修改而保留其他已验证结果。
_Avoid_: 全部回退、忽略失败

**阅读负荷**:
一个显示侧的实际阅读速度相对于其舒适阅读目标的比例，用于跨语言比较字幕速度。
_Avoid_: 原始 CPS、字符密度

**硬超速负担**:
连续字幕中超出硬阅读容量的文字负担，而不只是发生硬超速的字幕段数量。
_Avoid_: 硬警告数、超速率

**速度离散度**:
一个观看范围内典型快速字幕与典型慢速字幕之间的阅读负荷差异。
_Avoid_: 平均 CPS、速度方差

**相邻跳变指标**:
连续字幕边界中较突兀部分的相邻阅读负荷变化程度。
_Avoid_: 最大 CPS 差、单次跳变

**局部速度基准**:
从邻近主显示侧的稳健阅读节奏推导、并可由高可信语速证据有限修正的自适应目标基准。
_Avoid_: 局部平均 CPS、语速目标

**宽松预设**:
优先减少内容和时间轴改动、允许较高阅读速度的速度预设。
_Avoid_: 不处理模式、关闭优化

**均衡预设**:
在语义、同步、超速控制和节奏连续性之间采用默认折衷的速度预设。
_Avoid_: 普通模式、固定默认值

**平滑优先预设**:
采用更严格速度目标并允许受限双向平滑、优先改善连续阅读体验的速度预设。
_Avoid_: 强制统一模式、激进模式

**有效速度跳变**:
相邻显示侧之间同时具有足够阅读负荷和明显倍率变化、预计会被观众感知的速度跳变。
_Avoid_: 任意倍率变化、所有相邻差异

**紧急跳变上限**:
用于阻止少量极端相邻速度变化被整体分位指标掩盖的单边界限制。
_Avoid_: 最大 CPS、硬超速上限

**估算时间轴**:
在不读取真实媒体的情况下，根据已有字幕段边界和文本分布推导出的细粒度时间轴。
_Avoid_: 词级对齐、真实词时间

**对齐时间轴**:
将源语言字幕文本与对应媒体音轨进行匹配后得到、由真实语音证据支持的细粒度时间轴。
_Avoid_: 文本拆词时间、字幕轴优化

**关联媒体**:
用户为字幕任务可选提供的对应视频或音频，用于增强时间证据而不是作为字幕优化的必需输入。
_Avoid_: 必选视频、外部 VAD 文件

**媒体增强对齐**:
利用关联媒体和源语言字幕生成对齐时间轴的可选处理阶段；未启用时字幕任务继续使用估算时间轴。
_Avoid_: 默认重新转录、翻译文本对齐

**对齐质量检查**:
在接受对齐时间轴前，对源文本覆盖、时间顺序和媒体边界可信度进行的质量判断。
_Avoid_: 模型成功、非空时间戳

**对齐降级**:
媒体增强对齐不可用或未通过质量检查时，保留字幕任务并改用较低等级时间证据继续处理。
_Avoid_: 对齐失败终止、静默接受异常

**连续语音窗口**:
由明显停顿、受保护字幕或节奏重置点划分，可共同进行时间对齐和结构优化的一段连续语音范围。
_Avoid_: 整片音频、单个原始字幕段

**时间证据等级**:
根据对齐质量检查结果确定的时间证据可信程度，用于限制该窗口允许执行的结构调整范围。
_Avoid_: 成功或失败、模型置信度

**时间证据档案**:
与字幕和关联媒体内容绑定、保存对齐锚点、证据等级及生成条件的可复用时间证据记录。
_Avoid_: 缓存文件、私有字幕格式

**语义完整性**:
字幕经过压缩、重切或文本迁移后，原有事实、关系、语气、数字、专名、术语和信息顺序仍被保留的质量要求。
_Avoid_: 字面相似、主题相关

**语义复核**:
对确定性规则无法确认的字幕窗口进行独立判断，以决定接受修改、局部回滚或进入人工复查。
_Avoid_: 同模型自检、全量二次翻译

**变更记录**:
描述速度优化对字幕时间、结构和文字所作修改及其验证结果的可追踪记录。
_Avoid_: 调试日志、QA 摘要

**分析模式**:
只计算问题、候选修改和预期改善而不写入字幕结果的速度优化运行方式。
_Avoid_: 关闭优化、只读审计

**统一速度管线**:
集中完成速度分析、时间分配、结构调整、语义修复和质量报告的单一字幕处理管线。
_Avoid_: 旧功能组合、多个独立后处理开关

**速度配置方案**:
基于速度预设并包含用户高级覆盖项、可命名和复用的一组完整速度策略。
_Avoid_: 后处理配置方案、配置文件、临时自定义状态

**必要语义修复**:
确定性优化后仍需理解或改写内容才能解决的字幕窗口，应执行语义修复而不受人为调用预算限制。
_Avoid_: 无限调用、预算内候选

**可见字素**:
观众视觉上感知为一个字符的最小文字单位，组合字符和完整 Emoji 序列各视为一个单位。
_Avoid_: Unicode 码点、字符串长度

**分量式阅读负荷**:
混合文本按不同文字系统分别计算阅读需求后合并得到的整体阅读负荷。
_Avoid_: 主要语言分类、混合文本平均值

**边界移动额度**:
根据时间证据等级限制单个字幕边界能够偏离原位置的最大范围。
_Avoid_: 间隙上限、窗口时长

**尾部补偿**:
为避免字幕在停顿前过快消失，按其后间隙的时长为该段显示结尾补充的额外显示时长；补偿量随间隙单调不降并封顶。
_Avoid_: 尾部停留、tail dwell、lead-out

**补偿时长**:
尾部补偿实际为一段字幕结尾追加的显示时长，即补偿曲线在某个间隙处的取值。
_Avoid_: 停留时长、延长量

**最大闭合间隙**:
闪轴闭合与尾部补偿的共同边界：不超过它的间隙归闪轴闭合，超过它才进入补偿。
_Avoid_: 闭合间隙、flash 上限、补偿起始间隙

**最小补偿**:
间隙刚跨过最大闭合间隙时给予的补偿时长，即补偿区的下限，保证越过边界即有可感知的尾音。
_Avoid_: 补偿基线、起始补偿、最短停留

**最大补偿间隙**:
补偿时长达到上限的间隙；更大的间隙补偿不再增加，只让留白继续张开。
_Avoid_: 饱和间隙、场景切换间隙

**最大补偿**:
一段字幕结尾可获得的补偿时长上限。
_Avoid_: 停留上限、封顶值
