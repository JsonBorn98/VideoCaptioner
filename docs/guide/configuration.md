# 配置指南

VideoCaptioner 的设置按功能模块分开。字幕优化与翻译保留前置拆分、字数上限、优化和
翻译配置；独立的字幕后处理设置页管理阅读速度、时间轴、文本清理和质量审计。

## 字幕后处理方案

设置页内置三个可编辑模板：

| 模板 | 定位 |
|------|------|
| 宽松 | 尽量少改动，允许较大的局部速度变化 |
| 均衡 | 在改动幅度、连续性和阅读舒适度之间折中，完整流程默认使用 |
| 平滑优先 | 更积极地重组字幕，使整体显示速度更连续 |

页面修改会立即保存。每个内置模板都有不可变的出厂基线，可以随时恢复；新增方案必须
从一个内置模板复制，恢复时回到它所继承模板的出厂基线。每个具体参数也有独立的恢复
图标，不需要额外确认保存。

## 模块开关与默认行为

完整 workflow 默认执行字幕后处理，可在任务中关闭。独立后处理页面只有在用户启动
任务时才运行；方案中的能力可分别关闭。默认方案以译文阅读体验为主，原文主要作为
理解参考，也可以开启“双侧优化”同时处理两侧。

主要设置类别包括：

- 文本清理：占位符、引号和末尾弱标点。
- 时间轴：微小间隙修复、最短/最长显示时长和速度连续性。
- 阅读速度：中日韩字符与拉丁文字的舒适/硬 CPS 阈值。
- 语义修复：确定性调整不足时，使用受校验约束的局部 LLM 修复。
- 精准时间：可选关联视频或音频并调用 ForcedAligner；默认仍使用固定算法。
- 审计：参考侧检查、QA 报告和可复用时间证据。

ForcedAligner 不要求用户额外提供 VAD 文件。开启精准时间后，软件从关联媒体生成时间
证据；模型不可用或局部失败时会降级到固定算法。仅处理普通 SRT 时无需上传媒体。

完整字段和算法约束见[字幕速度优化设计规格](/dev/subtitle-speed-optimization-spec)。CLI
可使用 `videocaptioner config show` 查看当前配置，并用
`videocaptioner postprocess --help` 查看任务级覆盖参数。

## SRT 与样式

SRT 是模块间唯一的默认工作格式。导入 ASS/VTT 时，样式、定位和特效会被有意丢弃；
处理模块只保留普通字幕的文本、时间轴和单双语语义。需要 ASS 时，从后处理结果执行
“导出”，由字幕样式模块应用当前样式。

特效 ASS 不属于软件处理范围，请在导入前自行排除。更多操作说明见
[字幕后处理](/guide/subtitle-postprocessing)。
