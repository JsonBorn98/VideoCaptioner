# 使用 SRT 作为字幕模块间的唯一默认交付格式

字幕优化翻译、字幕后处理和其他内容处理模块之间统一交付 SRT，使各阶段只关注时间轴、文本和单双语语义，不承担 ASS/VTT 的表现层往返。外部 SRT、VTT 或 ASS 导入时均规范化为内容工作稿，原文件字体、颜色、分辨率、定位和特效被有意丢弃；处理完成默认输出新的 SRT。后处理页面继续保留可编辑结果工作稿，完成时自动保存一次 SRT，后续人工修改则由用户显式保存或导出。只有用户导出或视频合成需要呈现成品时，字幕样式模块才依据软件当前布局与样式配置生成 ASS 等格式。这放弃了第三方 ASS 无损往返，但消除了模块处理过程隐式继承、误改或伪装保留输入样式的歧义。

独立 GUI 若无法可靠识别单双语结构，只生成可检查预览，用户确认结构前不得开始强介入后处理；完整 workflow 则信任上游结构，非交互 CLI 明确警告后采用保守的默认角色解释。

软件只承诺导入一般纯文字字幕。不会建立第三方 ASS 内容轨道选择器，也不解释标牌、OP/ED、卡拉 OK、绘图、定位或其他高级事件的业务含义；用户须在导入前提供适合内容处理的普通文本字幕。解析器可以自然忽略非 Dialogue 事件并剥离表现标签，但不以此构成高级 ASS 兼容承诺。

字幕优化翻译页和字幕后处理页统一区分“保存”与“导出”：保存只提交当前工作稿到任务的规范 SRT 主文件；导出从当前工作稿生成 SRT、VTT 或 ASS 副本，其中 ASS 由字幕样式模块渲染。派生导出不替换模块间活动字幕，也不得覆盖最初导入的源文件。

完整 workflow 对每个产生字幕内容的阶段都执行不可关闭的 SRT 自动保存。workflow 编排页只提供一个阶段自动导出开关；启用后，转录、字幕优化翻译和字幕后处理等每个实际完成的字幕阶段都按任务开始前冻结的同一格式、布局和样式各生成一份派生成品，并通过阶段文件名前缀区分。模块线程本身仍不得直接生成 ASS 等表现成品。由此，自动导出可以满足无人值守交付，但不会让表现格式重新成为模块间契约，也不会引入逐阶段配置矩阵。

阶段自动导出只选择一个附加观看格式：ASS 或 VTT，默认 ASS。SRT 已由阶段自动保存提供，不重复列入自动导出；JSON 和 TXT 只保留为页面手动导出格式。选择 ASS 时冻结布局、样式方案和设计基准分辨率，选择 VTT 时样式配置不参与任务。

阶段文件统一使用 `【转录字幕】`、`【初版字幕】`、`【后处理字幕】` 三种前缀；同阶段的 SRT 与 ASS/VTT 派生成品共享前缀并按扩展名区分。完整 workflow 在任务创建时确定稳定基础名称，各阶段路径直接派生；独立导入只在建立基础名称时规范化一次既有阶段前缀。既有翻译服务等内容来源后缀可以保留在基础名称中。

SRT 是字幕内容持久化、独立调用和跨进程边界的唯一默认格式，不是同一完整 workflow 内相邻阶段必须执行的文件传输协议。连续阶段直接传递规范字幕数据快照，同时每阶段不可关闭地保存独立 SRT 检查点；下游不得仅为交接而重新解析刚写出的 SRT。

阶段 SRT 只保存用户可见成果，不承担软件重启后的 workflow 自动恢复。退出或崩溃后，用户若要继续，只能手动导入某一阶段 SRT 并重新调用所需模块；本次不增加任务 manifest、恢复入口或自动续跑状态机。
