# 将强介入字幕后处理设为独立可选模块

字幕优化翻译以可直接交付的初版字幕结束，速度优化、标点调整和其他显示质量修复归属于其后的独立字幕后处理模块。完整翻译 workflow 默认调用该模块，但保留初版字幕并允许用户在任务级关闭；独立后处理入口只在用户主动执行时运行。后处理也可以接收外部成型字幕，启用的能力允许大幅调整字幕结构、时间轴和必要文字表达，但必须生成独立的后处理字幕，不得回写输入字幕。这一边界同时保留了上游翻译与断句的稳定职责，以及下游解决严重超速问题所需的强介入能力。

默认组合开启均衡速度优化、必要语义修复和行尾弱标点清理；内容风格清理、独立间隙修复和媒体对齐保持显式选择。修改提交所需的内部质量验收不可关闭，额外报告文件则保持可选。

局部处理失败只回滚对应窗口。若模块整体无法生成合法后处理字幕，完整 workflow 明确告警并改用初版字幕继续下游流程；只有初版字幕本身无效时才终止。

完整 workflow 分别保存 SRT 格式的 `【初版字幕】` 与 `【后处理字幕】`；后处理成功时后者成为活动字幕输出，关闭或失败时前者成为活动字幕输出。独立任务无论输入格式为何，都在输入文件旁生成新的 `【后处理字幕】<名称>.srt`，所有路径都禁止原地覆盖输入。ASS 等样式化成品只由后续字幕样式导出生成。

完整任务与批量任务是否经过后处理，由任务创建或批量配置这一流程编排层选择，默认开启。字幕优化翻译等功能页面不展示其他模块的控制或设置入口。
