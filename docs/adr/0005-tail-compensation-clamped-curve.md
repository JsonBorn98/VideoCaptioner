# 尾部补偿采用单调钳制补偿曲线

字幕在停顿前过快消失时，为该段结尾补充显示时长（尾部补偿）。补偿量按其后间隙时长确定，形状为一条钳制曲线：不超过「最大闭合间隙」的间隙不补偿（归闪轴闭合），跨过该阈值即给「最小补偿」，随间隙线性上升，到「最大补偿间隙」处触顶「最大补偿」，更大的间隙补偿不再增加、只让留白继续张开。硬约束 `最大补偿 − 最小补偿 ≤ 最大补偿间隙 − 最大闭合间隙`（斜率 ≤ 1）保证补偿区内补偿量与留白均随间隙单调不降、永不重叠。

## Considered Options

- **离散分档（短/长两档定额）**：在档位跳变处会让留白突然变小（更大的间隙反而更挤），无法保证单调，弃用。
- **场景切换硬截断**（间隙超过某值即完全不补偿）：会造成"更大停顿、更短停留"的倒挂断崖；且纯字幕模式无法可靠判定真实镜头切换，间隙时长是很差的代理。弃用，改为让补偿量在「最大补偿」处饱和 —— 大间隙自然得到有界停留 + 大留白。

## Consequences

- 曲线由 4 个数值定义（两拐点坐标）：`最大闭合间隙`、`最小补偿`、`最大补偿间隙`、`最大补偿`；其中 `最大闭合间隙` 与闪轴闭合共用同一旋钮，故尾部补偿本身只新增 3 个参数。
- **最小留白不再是独立参数**，恒等于 `最大闭合间隙 − 最小补偿`；只要 `最小补偿 < 最大闭合间隙` 即恒正。
- 恰在「最大闭合间隙」处，留白有一个等于「最小补偿」的下跳，是保证最小尾音的刻意例外；单调性精确表述为"补偿区内单调"。
- UI 需就斜率 ≤ 1 与 `最小补偿 < 最大闭合间隙 < 最大补偿间隙` 联动钳制，用户无法配出重叠或倒挂曲线。
