# 策划编码器目录 + 能力探测（而非纯动态或纯静态）

## Status
accepted

编码器及其 preset/微调/快速解码/配置/级别与固定品质参数范围，来自一份策划维护的**"每编码器选项目录"**（Handbrake 式数据表），是 UI 选项渲染与 ffmpeg 参数映射的权威来源。某编码器在当前 ffmpeg 核心 + 硬件上能否实际运行，由**能力探测**判定：启动解析 `ffmpeg -encoders` 判编译支持，硬件编码器在"可用性测试"时做真实初始化探测；不可用项置灰并提示原因。

因捆绑的 essentials 构建只含编码器子集（例如**无 SVT-AV1**、libaom 慢、硬件编码依 GPU/驱动），能力探测是**必需**而非可选——它直接决定用户开箱能选哪些编码器。

## Considered Options
- **纯动态解析 `ffmpeg -h encoder=<name>`**（拒绝）：输出为自由文本、各构建不一、profile/level 常列不全，解析脆。
- **纯静态列表、编码时才失败**（拒绝）：用户会选到跑不了的编码器，UX 差。
- **混合：策划目录 + 探测可用性**（采纳）：UI 结构可预测，可用性准确，优雅应对换核心/换显卡。

编码器原生刻度：固定品质数值按各编码器映射其原生参数与范围（x264/x265/nvenc 0–51、AV1 0–63…），不强行统一成假的 RF 刻度。
