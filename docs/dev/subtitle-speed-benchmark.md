## 长序列性能冒烟

使用 5,000 cue 合成长序列验证长片复杂度。每 17 个 cue 注入一个 650ms 的中文硬超速尖峰，
其余 cue 为 1,050ms 正常字幕；均衡方案、译文主侧、无媒体证据。

| cues | elapsed | hard before | hard after | timing changes |
| ---: | ---: | ---: | ---: | ---: |
| 5,000 | 1.071s | 295 | 0 | 592 |

长序列使用整阶段事务验收：候选线性生成，最终 M3 不改善则整批回滚。1,201 cue 回归用例固定验证
该路径不劣化 M3。时间值来自本地开发机，仅用于防止复杂度退化，不作为跨设备性能承诺。

结论：

- 有安全空隙时可以消除 fixture 中的硬超速，并同时降低速度离散或相邻跳变。
- 没有安全空隙、完整修复超出低可信移动额度时保持原时间，不以累计漂移换取表面改善。
- 后续发布基准仍需加入真实长片、双语译文主侧、ForcedAligner 各质量等级和 LLM 语义修复。
