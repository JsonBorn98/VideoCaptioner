# 字幕后处理与速度优化技术规格

状态：首版实现完成。速度核心、独立后处理模块、统一字幕 I/O、GUI/CLI 入口和 workflow
调用边界均已落地，进入真实任务校准阶段。

## 1. 背景

当前字幕后处理中的阅读速度能力由三个彼此独立的功能组成：逐段 CPS 审计、中文硬超速 LLM 压缩、字幕间隙闭合。它们只能识别或修复单段问题，不计算全片速度离散度和相邻速度跳变，也不感知最终字幕布局。

真实任务显示，译文通常不是整体过快，而是大量较慢字幕中夹杂少量极快尖峰。当前功能无法让这些字幕形成平稳连续的局部阅读节奏。

字幕优化翻译中的拆字、智能断句和中英文单句上限属于上游内容生产流程。它们负责生成一份
可以直接交付的初版字幕，本次字幕后处理模块化不改变这些行为。字幕后处理只接收上游初版字幕
或外部成型字幕，并生成独立结果；它可以再次重排结构，但不得回写输入版本。

## 2. 目标

1. 消除可安全修复的硬超速，并明确报告未解决超速。
2. 降低全片速度离散度和连续字幕的相邻速度跳变。
3. 默认以翻译文本体验为主，不因参考原文过快而擅自改写源字幕。
4. 允许在可信边界内调整时间、重新切分、合并或迁移文本。
5. 纯 SRT 输入必须完整可用；关联视频或音频时可选增强时间对齐。
6. 提供开箱即用预设，同时尽量暴露高级参数、命名方案和分层恢复默认能力。
7. 所有自动修改必须可审计、可回滚、可追踪，并通过其风险等级规定的验证门槛。
8. 同时支持完整 workflow 的可选下游阶段，以及直接输入成型字幕的独立调用方式。
9. 单语和双语字幕使用明确的显示侧分类，不因缺少译文侧而拒绝处理。

## 3. 非目标

1. 不把译文直接与源语言音频做强制对齐。
2. 不为了满足硬限静默丢失事实、数字、专名、术语或语义关系。
3. 不把 VAD 当作词级对齐；VAD 只提供语音与停顿范围。
4. 不要求字幕优化任务必须提供媒体文件。
5. 不保留旧阅读速度审计、快速字幕压缩和闪烁修复的独立兼容管线。
6. 不提供用户侧 token、窗口数或费用预算。调用合理性由候选过滤、缓存和有限重试保证。
7. 不接管或改变上游字幕优化翻译的拆字、断句、单句上限和翻译职责。

## 4. 模块边界与 workflow

```text
字幕优化翻译 -> 初版字幕 -----------------------> 直接使用
                  |
                  +-> 可选字幕后处理 -> 后处理字幕

外部单语/双语成型字幕 -> 独立字幕后处理任务 -> 后处理字幕
```

- 初版字幕和外部成型字幕都是不可回写的输入快照。
- 字幕后处理是独立任务类型，不是字幕优化翻译线程中的隐式收尾步骤。
- 用户启用的后处理能力可以强介入字幕结构、时间轴、内容归属和必要文字表达。
- 用户可以关闭整个后处理模块，也可以只关闭其中某些能力项。
- 完整翻译 workflow 默认启用后处理，用户可在任务级关闭；独立入口只在用户主动执行时运行。
- 精准对齐、速度优化、标点规范化、占位符清理、间隙处理和报告均归属于字幕后处理。
- 上游输出与后处理输出必须具有不同的文件身份，失败或回滚不得损坏输入字幕。
- 完整 workflow 使用上游明确的单双语布局元数据，不重新猜测字幕结构。
- 外部字幕优先读取可信布局标记，否则自动识别；低置信度时显示可检查预览和警告，用户须明确选择单语、原文在上或译文在上。
- 局部窗口失败只回滚该窗口；模块级失败回退初版字幕并继续完整 workflow。只有初版字幕本身也无效时才终止。

## 5. 核心产品决策

### 5.1 优化范围

- 采用结构级速度优化，可调整字幕段边界、段数和文本归属。
- 默认使用译文主显示侧驱动速度目标。
- 可选模式同时审计参考显示侧硬限，但默认不改写参考侧文本。
- 结构调整必须保持原文和译文的顺序与对应关系。
- 单语字幕将唯一显示侧作为主侧，执行所有已启用能力。
- 双语字幕的标点清理、内容压缩和语义修复默认只修改译文；拆分、合并、内容归属和时间轴调整必须同步维护两侧映射。
- “同时优化两侧”是显式选项，开启后文本类能力可分别作用于原文与译文，但不得破坏对应关系。

### 5.2 速度目标

- 使用局部自适应目标，不追求全片固定 CPS。
- 局部基准以主显示侧的稳健、时长加权中位阅读负荷为主。
- 存在高可信词时间或 VAD 时，语速密度只对局部基准做有限修正。
- 受保护字幕、硬超速尖峰和节奏重置点不参与局部基准估计。
- 默认单向平滑，只降低快速尖峰。
- 平滑优先方案允许受限双向平滑，但不缩短受保护字幕。

### 5.3 冲突优先级

默认优先级：

1. 语义完整性和事实正确性。
2. 不跨越可信语音边界、受保护字幕和节奏重置点。
3. 消除硬超速。
4. 降低相邻跳变和全局离散度。
5. 减少字幕结构改动。

严格硬限是显式可选策略，可以提高文本压缩强度，但不能绕过最低语义底线。无法安全解决的硬超速进入 QA。

## 6. 阅读负荷模型

### 6.1 可见字素

- 使用 Unicode grapheme cluster，而不是 Python 字符串长度。
- 完整 Emoji 和组合字符各计为一个可见字素。
- 空白默认权重为 0。
- 标点使用可配置权重。

初始标点权重建议：

| 类型 | 初始权重 |
| --- | ---: |
| 空白 | 0.0 |
| 弱标点，如逗号、句号 | 0.25 |
| 强标点，如问号、感叹号、省略号 | 0.5 |
| Emoji / 图形符号 | 1.0 |

这些权重属于首次实现默认值，必须通过真实样本基准校准。

### 6.2 分量式阅读时间

混合文本不再按主要语言二选一。对每个显示侧分别统计 CJK 和非 CJK 阅读单位：

```text
required_comfort_seconds =
    cjk_units / comfort_cps_cjk
  + latin_units / comfort_cps_latin

required_hard_seconds =
    cjk_units / hard_cps_cjk
  + latin_units / hard_cps_latin

comfort_load = required_comfort_seconds / actual_duration_seconds
hard_load = required_hard_seconds / actual_duration_seconds
```

`hard_load > 1` 表示硬超速。

首版分类契约：

- `cjk_units` 包含 Han、Hiragana、Katakana、Hangul 字素。
- 其他字母、数字及未单列文字系统统一计入 `non_cjk_units`；配置字段仍沿用
  `*_latin` 仅作为面向用户的历史命名，核心模型使用 `non_cjk`。
- Emoji 和图形符号按一个非 CJK 字素计数，标点只应用标点权重，不重复计数。
- 使用 `regex` 的 `\X` 切分，并固定依赖版本；相同输入按 cue 稳定 ID 排序。
- `duration <= 0` 的 cue 标记为 invalid，不参与分位数或相邻倍率，并必须进入 QA。
- 空白或空文本 cue 的 required time 和 load 均为 0；不参与倍率分母。

### 6.3 组合质量指标

#### 硬超速负担

```text
HardDeficit =
sum(max(0, required_hard_seconds - actual_duration_seconds))
-----------------------------------------------------------
sum(required_hard_seconds)
```

逐段警告继续用于定位，但不能再用警告数量代表总体严重程度。

#### 速度离散度

在同一节奏区间内，以显示时长为权重计算阅读负荷的 P10 和 P90：

```text
SpeedSpread = weighted_P90(comfort_load) / weighted_P10(comfort_load)
```

#### 相邻跳变

仅比较未跨节奏重置点、双方均有有效主显示文本的相邻字幕：

```text
jump_ratio = max(load_i, load_i+1) / min(load_i, load_i+1)
```

只有较快一侧达到可配置的有效负荷门槛时，该倍率才计入有效速度跳变。首次实现建议门槛为 `0.6`，需通过基准校准。

同时控制：

- 有效跳变的 P90。
- 单边界紧急跳变上限。

### 6.4 首版统计契约

- 加权分位数：按值升序、稳定 ID 破同值，取累计正权重首次达到
  `q * total_weight` 的观测值；权重为合法显示时长。
- 少于 3 个正负荷样本时 `SpeedSpread` 为 `null`；P10 为 0 时同样为 `null`，不输出
  无穷大或 NaN。
- `AdjacentJump` 是 `{p90, emergency_count, compared_count}`，不是单一浮点数；少于
  2 个有效边界时 P90 为 `null`。
- `HardDeficit` 在 required time 总和为 0 时为 0。
- 局部基准默认取同一节奏区间内、目标 cue 前后各 4 个合法 cue 的时长加权中位数；
  排除受保护项、hard load > 1 的尖峰和零负荷项。少于 3 项时依次回退到节奏区间中位数、
  全片中位数、当前预设 comfort load 1.0。语音密度修正首版限制在基准的 +/-15%。

## 7. 内置预设

| 参数 | 宽松 | 均衡（默认） | 平滑优先 |
| --- | ---: | ---: | ---: |
| CJK 舒适 / 硬限 CPS | 10 / 12 | 9 / 11 | 8 / 10 |
| 外文舒适 / 硬限 CPS | 18 / 22 | 16 / 20 | 14 / 18 |
| P90 相邻倍率目标 | 2.2x | 1.8x | 1.5x |
| 单边界紧急上限 | 3.5x | 3.0x | 2.5x |
| 普通最短时长 | 0.8s | 1.0s | 1.2s |
| 普通最长时长 | 7.0s | 6.0s | 5.5s |
| 双向平滑 | 关闭 | 关闭 | 开启 |

全局技术最短时长初始值为 `0.5s`。受保护字幕可以合法超过普通最长时长。

所有参数均有权威默认值，并允许高级覆盖。

## 8. 节奏区间与时间调整

### 8.1 节奏重置

- 有词时间或 VAD 时，根据真实停顿和局部讲话节奏判断。
- 只有字幕时间轴时，回退阈值默认 `800ms`。
- 超过硬重置值 `1500ms` 时始终开始新的节奏区间。
- 节奏重置阈值与闪烁/间隙闭合参数完全分离。

### 8.2 时间证据模型

时间证据必须分别保存 `provenance`、`granularity`、`coverage`、`quality_grade` 和
`allowed_operations`。来源不自动等于质量；ForcedAligner 结果也必须经过窗口级质量检查。
锚点可拥有比窗口更低的局部质量等级。

现有 `is_word_timestamp()` 只能描述文本外观，不得再作为可信度来源。

### 8.3 边界移动额度

初始值：

| 证据 | 单边界最大偏移 |
| --- | ---: |
| 仅 SRT / 估算时间轴 | 250ms |
| VAD / 中可信对齐 | 500ms |
| 高可信 ForcedAligner | 1000ms，优先吸附词边界 |

任何等级都不得跨越受保护字幕或节奏重置点。数值支持高级覆盖。

额度始终相对任务输入快照中的原始边界计算，不因多轮候选累加。首版操作矩阵：

| 质量 | 允许操作 |
| --- | --- |
| 低 / 纯字幕 | 占用原 cue 相邻空隙、在 250ms 内移动共享边界；不跨 cue 迁移文本 |
| 中 | 上述操作、按可靠停顿重切；只允许同一语义窗口内的保序迁移 |
| 高 | 上述操作、按词锚点重切、合并和边界重建 |

“安全空隙”是相邻非重叠 cue 之间、未与保护范围、重置点或可靠语音区间相交的时间。

## 9. 受保护字幕

显式保护规则优先于自动识别。支持：

- ASS 样式或标签。
- 字幕索引或时间范围。
- 用户正则表达式。
- 用户手动锁定。

自动候选包括：

- 标题卡和章节标题。
- 歌词、音乐标记和片尾字幕。
- 短文本长显示。
- 长停顿两侧的刻意显示。

判断不确定时不缩短，并进入 QA。

保护项的首版语义是：不得改写文本、缩短时长、移动自身边界、拆分或合并，也不得让其他
cue 的文本或边界跨越它。显式规则优先于自动规则；冲突时采用更强保护。用户正则必须限制
长度和执行时间，超限规则被禁用并进入设置诊断。

## 10. 媒体增强对齐

### 10.1 默认行为

- 字幕后处理模块默认只需成型字幕，继续使用字幕自身时间轴作为低等级时间证据。
- 用户可选关联视频或音频。
- “精准时间轴”开关默认关闭。
- 视频输入先提取选定音轨，再调用 ForcedAligner。
- ForcedAligner 对齐源语言字幕与源音频，译文只映射结果。
- 对齐输入使用完成文本清理和源字幕校正后的规范源文本；其指纹变化会使旧证据失效。
- 只有译文、缺少源侧、语言未知或音轨未选择时，精准时间轴不可用并在 UI 显示原因。

### 10.2 UI 触发

采用 F3：任务开始时先运行对齐和质量检查。正常窗口自动继续；异常窗口显示摘要。失败默认局部降级，不阻塞整项任务。

### 10.3 对齐窗口

- 先按 cue gap 和快速 VAD 粗分窗口，再对齐并按质量结果细分；不整片一次对齐，也不锁死为逐 cue 对齐。
- 窗口不得跨越受保护字幕或硬节奏重置点。
- 初始工程默认：目标窗口约 `45s`、最大 `90s`、音频余量 `1.5s`。这些值必须通过性能与准确性基准校准。
- 异常窗口可以逐 cue 重试；仍失败则回退估算时间轴。

### 10.4 对齐质量分级

质量检查至少包含：

- 源文本覆盖率。
- 未匹配文本比例。
- 时间戳单调性。
- 音频范围越界。
- 对齐跨度与窗口跨度的一致性。
- 重复、截断和病理性密度。

使用方式：

- 高可信：允许词级重切、文本迁移和边界重建。
- 中可信：只使用可靠停顿和大致语音范围，限制跨段迁移。
- 低可信：完全回退估算时间轴。

## 11. 字幕后处理管线

旧阅读速度审计、快速字幕压缩和闪烁修复最终由以下单一管线替代：

```text
接收不可回写的成型字幕或上游内存快照并规范化为内容工作稿
-> 文本清理与保护识别
-> 读取或生成时间证据
-> 主显示侧解析
-> 阅读负荷与局部基准分析
-> 确定性结构和时间优化
-> 必要语义修复
-> 候选语义验证
-> 最终时间分配
-> 时间与结构验证
-> M3 指标复审
-> 原子提交或窗口回滚
-> 输出 SRT 模块交付字幕、QA、变更记录和可选时间证据档案
```

翻译、上游拆字、智能断句和单句上限不属于该管线。标点规范化不能再在翻译或字幕优化完成后
隐式执行；只有字幕后处理任务启用对应能力项时才执行。

### 11.1 确定性阶段

按风险由低到高尝试：

1. 利用安全空隙。
2. 在移动额度内重新分配相邻边界。
3. 按词锚点、标点和语义边界重切。
4. 合并极短碎片或重新分配窗口内文本归属。
5. 平滑优先方案中，受限缩短明显偏慢且未受保护的字幕。

候选按不可重叠写窗口生成；相邻 cue 作为只读 halo。提交时校验窗口版本和共享边界所有权，
冲突候选重新分析，不覆盖已提交结果。候选快照同时用于分析模式和应用模式。

唯一接受谓词（浮点容差 `1e-9`）：

1. HardDeficit 不增加，未解决硬超速数量不增加。
2. 紧急跳变数量不增加。
3. SpeedSpread 与 AdjacentJump P90 均不得恶化超过 1%；其中至少一项改善超过 1%，或
   HardDeficit 有严格改善。
4. 所有行为不变量和该风险等级的语义门槛通过。

比较先在写窗口加 halo 上执行，再对整片执行；任一层失败都拒绝候选。

### 11.2 语义修复阶段

- 只处理确定性阶段仍无法解决的必要窗口。
- 默认语义窗口约 5 个字幕段，可配置；不能跨保护或重置边界。
- 可以在窗口内移动文本、重切或合并，但保持信息总量、顺序和时间范围。
- 不提供用户侧窗口或 token 预算。
- 防浪费措施：候选过滤、窗口去重、非重叠批处理、结果缓存、有限反馈重试。
- 每个窗口最多 2 次反馈重试；任务支持取消、断点续跑、并发限制和服务端速率限制。
- 不设置用户可见的费用预算或窗口配额，但任务开始前显示将使用远程 LLM；所有必要窗口均会
  处理，失败只将该窗口标记为 `unresolved`，不会静默跳过。
- 失败只局部回滚。状态为 `accepted / review_required / rolled_back / unresolved`。

## 12. 语义验证

采用 N3 分层验证。

### 12.1 始终执行的确定性检查

- 非空、时间单调、无非法重叠和媒体越界。
- 窗口源文本与结果覆盖完整。
- 数字、日期、百分比、金额和单位。
- 专有名词、用户术语、代码、URL 和公式。
- 否定关系和明显信息顺序。
- 段结构与时间证据等级允许的操作范围。

### 12.2 独立语义复核

仅在以下情况触发独立请求：

- 压缩率高。
- 关键实体或数字发生变化。
- 字面相似低但可能是合法紧凑改写。
- 严格硬限导致较强改写。
- 确定性检查无法判断窗口信息覆盖。

首版不强制安装 sentence-transformers 或 NLI 模型。复核可使用独立提示词和独立请求；高级配置允许指定不同复核模型。复核不可用时，高风险窗口保守回滚。

## 13. 配置与用户界面

### 13.1 默认启用

- 完整翻译 workflow 默认启用字幕后处理并生成独立的后处理字幕，同时保留初版字幕。
- 用户可以在任务开始前关闭整个后处理阶段，关闭后直接使用初版字幕。
- 独立后处理任务只在用户主动执行时运行。
- 默认能力组合：均衡速度优化、必要语义修复、行尾弱标点清理开启。
- 引号规范化、占位符清理、独立间隙修复、精准时间轴、严格硬限、双向平滑和参考侧优化关闭。
- 内部质量验收始终执行；额外 QA 报告文件默认关闭。
- 缺少 LLM 时仍执行确定性阶段，并将无法完成的必要语义修复窗口列为未解决项。

### 13.2 设置入口

主导航在“字幕优化翻译”之后提供独立“字幕后处理”页面。该页面同时承担成型字幕输入、任务
执行、结果预览和常用能力开关；完整高级设置通过页面内设置入口进入字幕后处理设置子页面。

具体导航归属：

- 主页 workflow 分段栏按“任务创建 → 语音转录 → 字幕优化翻译 → 字幕后处理 → 字幕视频合成”排列。
- 手动切换到字幕后处理页即可创建独立任务。
- 删除主导航中的独立“字幕速度”入口。
- 全局设置建立“字幕后处理”专属类别或子页面，承载所有高级设置。

字幕优化翻译页面不得保留后处理总开关、方案摘要、速度快捷项、媒体关联或后处理设置入口。
每个功能页面只关注自身模块；完整 workflow 的阶段选择由流程编排层负责。

任务创建页作为完整 workflow 的编排层，默认选择字幕后处理阶段，并允许用户为本次任务取消。
批量处理页在批量任务配置中提供相同阶段选择。该选择只决定是否经过后处理，不承载各能力项的
详细设置。

设置子页面按能力分为：

1. 概览与能力项。
2. 文本与标点。
3. 阅读速度目标。
4. 时间轴与结构。
5. 语义修复。
6. 精准对齐。
7. 报告与存储。

页面保持工作型、可扫描布局，不使用大型装饰卡片。普通参数直接展示，高风险和低频参数按分组展开。

### 13.3 后处理任务控制

字幕后处理页面使用紧凑任务控制：

- 选择单语或双语成型字幕；布局可自动识别并允许人工修正。
- 自动识别结果显示为“单语 / 原文在上 / 译文在上”；独立 GUI 的低置信度输入可以载入工作稿预览，但用户明确确认结构前禁用开始处理，不能静默拆分文本行。
- 完整 workflow 信任上游结构；带有可信 `SubtitleLayout` 元数据的 VideoCaptioner ASS 信任该元数据。
- 非交互 CLI 的低置信度输入明确警告并采用保守的默认角色解释，用户可通过 `--layout` 消除歧义。
- 启用或关闭各后处理能力项。
- 选择宽松、均衡、平滑优先或自定义后处理配置方案。
- 应用优化 / 仅分析。
- 关联媒体。
- 精准时间轴开关，仅有关联媒体时可用。
- 设置入口。

页面支持两种任务状态：

- 编排模式：完整 workflow 进入该阶段后自动运行冻结的配置，完成后自动进入视频合成；页面只显示本阶段进度、结果和取消操作。
- 独立模式：用户选择成型字幕、确认结构和能力项后手动开始，不触发任何上游或下游功能。
- 两种模式复用同一任务模型、核心管线和结果组件，不维护两套后处理实现。

页面复用字幕优化翻译页的字幕表格与格式导出能力，但工作稿和任务状态归字幕后处理模块所有：

- 导入字幕规范化后进入可编辑表格，用户可在处理前核对内容和单双语结构。
- 后处理从开始时的表格快照运行，完成后将结果回填为新的可编辑工作稿。
- 完成时自动原子保存一次 `【后处理字幕】<名称>.srt`，并将工作稿标记为已保存。
- 完成后的人工编辑只改变工作稿并显示未保存状态，不逐单元格自动覆写磁盘文件。
- 用户通过保存或导出显式提交后续修改；ASS 导出调用字幕样式模块，不继承输入 ASS 样式。
- 保存只更新自动命名的规范 SRT 主文件；导出菜单生成 SRT、VTT 或 ASS 派生副本，不改变任务活动字幕路径。
- 字幕优化翻译页与字幕后处理页必须采用相同的保存/导出语义和共享组件，不能在两个页面中维护含义不同的同名操作。
- 完整 workflow 的每个字幕产出阶段都必须自动保存规范 SRT，该行为默认开启且不可关闭。
- workflow 编排页提供一个可选的阶段自动导出开关，任务开始时冻结统一的导出格式、字幕布局、样式方案和设计基准；启用后，每个实际完成的字幕阶段都由编排层调用共享导出器，不提供逐阶段开关或逐阶段样式矩阵。
- 阶段文件使用固定的 `【转录字幕】`、`【初版字幕】`、`【后处理字幕】` 前缀；同阶段的规范 SRT 和 ASS/VTT 派生成品共享前缀并按扩展名区分，输出文件数量不作为省略阶段导出的理由。
- workflow 在任务创建时确定稳定基础名称，各阶段路径直接由基础名称和阶段前缀派生；独立导入只在建立基础名称时规范化一次既有阶段前缀，翻译服务等内容来源后缀可以保留。
- 阶段自动导出只允许单选一个附加观看格式，选项为 ASS 或 VTT、默认 ASS；SRT 已由不可关闭的阶段自动保存提供，JSON/TXT 只在页面手动导出中保留。
- 选择 ASS 时冻结字幕布局、样式方案和设计基准分辨率；选择 VTT 时不读取或应用 ASS 样式配置。
- 模块线程返回规范字幕数据快照和阶段 SRT 检查点路径，不直接写 ASS/VTT 派生成品；同进程下游消费数据快照，只有恢复、独立调用或跨进程时才读取 SRT。
- 规范 SRT 检查点保存失败使该阶段失败；ASS/VTT 自动导出使用临时文件原子提交，失败只进入 workflow 告警摘要，不撤销阶段结果或阻止后续模块。
- 阶段 SRT 是成果文件而不是自动恢复清单；本次不持久化完整 workflow manifest，不提供软件重启后的自动续跑。用户需要恢复时手动导入阶段 SRT，并逐个重新调用后续模块。

### 13.4 默认值与恢复

采用 RS3：

- 每个偏离预设的设置项显示单项复位图标。
- 不设置重复的分组或整页重置按钮；每个设置项使用常见复位图标恢复权威默认值。
- 有覆盖项时状态显示“自定义”。
- 整页恢复前显示受影响设置数量。

### 13.5 后处理配置方案

- 宽松、均衡和平滑优先是三套完整后处理模板，不只是速度参数预设。
- 每个模板同时具有不可变出厂基线和用户可编辑工作值；单项或整体恢复使用本模板出厂基线。
- 自定义方案只能从三个模板之一复制创建，复制来源模板创建时的当前有效值，并保存 `base_template_id`。
- 自定义方案恢复默认时使用来源模板的不可变出厂基线，不跟随来源模板之后的用户修改。
- 模板不可删除或重命名；自定义方案可新建、重命名、复制和删除。
- 支持版本化 JSON 导入导出、来源模板记录和完整字段校验。
- 后处理页面修改能力项或参数时立即持久化到当前选中的方案，不建立任务级临时覆盖层。
- 任务创建时冻结所选方案的完整快照，之后修改方案不影响运行中的任务。

## 14. 时间证据持久化

采用 Y3：

- 默认写入应用内部缓存。
- 可选在字幕旁生成 `.vctiming.json`。
- SRT/ASS/VTT 保持标准格式，不嵌入私有标记。
- 档案包含字幕内容指纹、媒体内容指纹、音轨、模型、配置版本、词锚点、窗口证据等级和质量结果。
- 指纹不依赖绝对路径，移动项目后仍可复用。
- 指纹或版本不匹配时拒绝复用。
- 档案不保存音频内容。

## 15. 输出与交付

采用 U3：

- 高可信且通过验证的修改自动写入新输出字幕。
- 原始输入字幕始终保留。
- 外部 SRT、VTT、ASS 导入时只提取时间轴、文本和单双语角色；输入样式、定位和特效不进入后处理工作稿。
- 输入范围限定为一般纯文字字幕；第三方 ASS 多轨语义、标牌、歌词、卡拉 OK、绘图、定位和特效识别属于明确非目标，软件不提供内容轨道选择或 Aegisub 式工程编辑能力。
- 用户负责在导入前排除不希望参与翻译或显示优化的特效字幕内容；解析器自然忽略非 Dialogue 事件并剥离表现标签，但不承诺高级 ASS 的正确往返。
- SRT 是字幕内容持久化、独立调用和跨进程边界的唯一默认格式，不沿用输入扩展名；同一完整 workflow 的相邻阶段直接传递规范字幕数据快照，不为交接执行 SRT 写盘后重读。
- ASS 等呈现格式只在用户导出或视频合成需要时，由字幕样式模块根据软件当前配置生成。
- 完整 workflow 同时保留可直接使用的初版字幕；后处理结果使用不同输出路径。
- 分析模式不写入修改结果。
- QA 报告包含 M3 前后指标、未解决超速、受保护项、降级窗口和语义复查队列。
- 结构化变更记录包含每项时间、结构和文本修改及验证结果。
- 高风险窗口不阻塞其他窗口。
- 完整 workflow 的后处理模块级失败必须记录原因、回退初版字幕并继续下游任务，不能把失败结果伪装成后处理字幕。

首版输出：

```text
【初版字幕】<name>.srt                  # 完整 workflow 始终保留
【后处理字幕】<name>.srt                # 后处理成功时生成
<output>.qa.md
<output>.speed-changes.json
<output>.vctiming.json        # 可选
```

- 后处理成功时，视频合成等下游阶段使用后处理字幕。
- 后处理关闭或模块级失败时，下游使用初版字幕。
- 独立任务保留原输入，并在其目录旁路生成 `【后处理字幕】<原文件名>.srt`。
- 用户从结果工作稿导出 ASS 时，使用当前字幕布局、字幕样式和设计基准分辨率，不继承输入 ASS 的表现属性。
- 输出使用临时文件完成验证后原子提交；不得原地覆盖输入字幕。

## 16. 配置模型建议

核心使用版本化 `PostprocessProfile` 作为完整方案，并在其中组合单一 `SpeedPolicy`。三个后处理模板和
速度策略都由权威默认注册表提供，GUI、CLI 和任务对象不得各自复制默认值。
完整 workflow 的阶段选择与 profile 分离。配置解析优先级从低到高为：来源模板出厂基线、模板或
自定义方案持久化值、CLI 显式参数。GUI 不维护任务级临时参数覆盖。完整方案至少包含：

- preset / profile id。
- enabled / analyze-only。
- primary-side / reference-side policy。
- 各语言舒适与硬限。
- 可见字素和标点权重。
- 局部窗口和语速修正强度。
- M3 门槛。
- 节奏重置阈值。
- 普通时长范围。
- 各证据等级边界移动额度。
- 允许的结构操作。
- 保护规则。
- 语义窗口和复核策略。
- 精准对齐与模型配置。
- 报告、缓存和 sidecar 配置。

旧 `audit_reading_speed`、`compress_fast_subtitles`、`fix_gaps` 最终删除，但必须在新核心、分析模式、
QA、配置适配和迁移测试完成后才切换默认。首个版本读取旧设置并一次性映射到新 profile；速度优化
关闭时，原 gap repair 仍作为内部回退保留，直到统一管线覆盖等价行为。

## 17. 运行时数据建议

现有 `ASRDataSeg(text, translated_text, start_time, end_time)` 不足以支持该设计。新管线需要独立上下文保存：

- 最终布局和主显示侧。
- 稳定 cue/side/token/anchor ID，多对多原译映射及 split/merge lineage。
- 时间证据来源、粒度、覆盖率、窗口/锚点质量和允许操作。
- 词锚点与保护范围。
- 速度策略快照。
- 修改与回滚记录。
- 输入快照版本、候选版本、写集合、只读 halo 和提交状态。

建议引入 `SpeedOptimizationContext` 和 `TimingEvidenceBundle`，而不是继续把可信度猜测塞入 `is_word_timestamp()`。标准字幕文件解析仍返回 ASRData，上下文在任务内关联，并可通过时间证据档案跨任务恢复。

## 18. CLI 建议

提供独立 `postprocess` 命令；完整 workflow 只负责决定是否在初版字幕后调用它。速度参数属于
该命令和后处理配置，不再挂靠翻译命令：

```text
--speed-optimize / --no-speed-optimize
--speed-mode apply|analyze
--speed-profile NAME
--speed-profile-file PATH
--speed-primary translate|original|layout
--speed-reference-audit
--speed-media PATH
--speed-precise-timing
--speed-save-timing-sidecar
```

全部高级参数继续可通过配置文件覆盖，不要求为每个参数增加 CLI flag。

## 19. 验收标准

### 19.1 行为不变量

- 不产生非正时长、逆序段，或同一渲染轨道内非输入既有的重叠；ASS 不同 layer 的既有重叠保留。
- 不跨保护范围和节奏重置点。
- 参考侧默认不改写。
- 语义验证失败只回滚对应窗口。
- 对齐失败只降低该窗口证据等级。
- 确定性阶段对相同输入和策略可重复。
- 跳过后处理时，初版字幕不得因标点、间隙、速度或精准对齐配置而发生变化。
- 独立后处理任务不得触发翻译、上游拆字或智能断句。
- 后处理模块级失败时，完整 workflow 必须以初版字幕继续；初版字幕有效时不得因后处理失败整体终止。
- 对已优化结果连续运行 3 次，第二次以后文本和段数不变、任一边界漂移不超过 1ms、M3 差异
  不超过 `1e-9`。

### 19.2 指标验收

- 接受的修改不得增加 HardDeficit。
- 接受谓词统一采用 11.1 的局部加全局规则。
- 每个超过当前 profile 紧急上限的边界必须单独进入 QA；候选不得增加其数量。
- 未解决硬超速必须进入 QA。
- 分析模式报告与应用模式修改前基线一致。

### 19.3 覆盖范围

测试至少覆盖：

- 纯中文、纯外文、混合文字、Emoji、组合字符和标点权重。
- 单语、双语四种布局和译文缺失回退。
- 双语默认译文主侧、原文参考侧，以及显式同时优化两侧。
- 外部多行单语、同语种双语、短样本和低置信度结构识别降级。
- 0/负时长、重叠、长停顿、极短片段和受保护字幕。
- 三种预设、所有默认恢复层级和自定义方案导入导出。
- 单向与双向平滑。
- 所有时间证据等级和移动额度。
- ForcedAligner 高/中/低可信、缓存、sidecar 和降级。
- LLM 窗口重切、实体保护、独立复核、有限重试和局部回滚。
- 长视频窗口化、缓存命中、取消任务和部分失败。
- CLI、GUI 和核心配置默认值一致。
- 完整 workflow 的启用与跳过两条路径，以及独立单语/双语后处理入口。
- 任务创建与批量编排的后处理阶段选择，且字幕优化翻译页面不存在跨模块控制。
- 编排模式自动运行与续接、独立模式手动运行，且两种模式产出一致。
- workflow 分段顺序、独立页面入口、移除旧字幕速度导航和后处理设置类别。

### 19.4 实证基准

使用脱敏后的长序列 fixture 和现有真实任务做优化前后对比，至少记录：

- HardDeficit。
- CJK/外文硬超速段和未解决段。
- SpeedSpread。
- AdjacentJump P90 与紧急跳变数量。
- 结构修改数量、平均边界移动和最大边界移动。
- LLM 窗口数、验证重试数、回滚数和缓存命中率。
- ForcedAligner 各证据等级窗口占比。
- 总耗时和峰值内存/显存。

首次发布前必须用这些结果校准标点权重、有效跳变门槛、对齐窗口时长和三档移动额度。

## 20. 实施边界

用户已整体确认。本次实施采用可随时运行和回退的纵向切片：

1. 建立版本化策略、阅读负荷、M3、稳定 ID、分析快照和单元测试。
2. 交付纯字幕低可信模式：保守时间优化、QA、变更记录、分析/应用模式和旧配置适配。
3. 建立独立字幕后处理任务、页面和设置页；从翻译线程移除隐式标点与速度收尾。
4. 建立时间证据 schema、缓存/sidecar 原子写和 ForcedAligner 契约 fixture，再接媒体增强。
5. 接入高/中可信结构操作、语义窗口事务、LLM 修复和分层验证。
6. 完成 profile 导入导出、CLI、格式能力矩阵、真实样本与长视频基准。
7. 达到回归门槛后将完整翻译 workflow 默认接入后处理；旧速度步骤不与新后处理路径叠加。

默认路径不调用 ForcedAligner；用户显式关联媒体并开启精准时间轴时，用证据质量决定
250/500/1000ms 共享边界额度。语义修复默认随翻译速度优化启用，但缺少 LLM 模型时只执行确定性
阶段。后续版本不得静默改变本节已公开的策略和指标序列化契约，只能通过 schema 版本升级扩展。
