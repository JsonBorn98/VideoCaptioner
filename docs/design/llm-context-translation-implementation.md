# Enhanced LLM Translation Implementation Specification

状态：需求已确认，作为 [LLM Context Translation Design](./llm-context-translation.md) 的实现细则。

本文只承载技术与界面细节；产品工作流、默认行为和模块边界以主设计为准。

## 全文分析与术语处理

- 全文分析窗口按 token 预算切分，切点落在字幕段边界，相邻窗口保留少量重叠；窗口输出携带原始字幕编号，汇总时消除重叠并去重候选。
- “全文”表示覆盖全部源字幕，不要求一次请求装入全文。超长内容递归汇总为翻译上下文简报。
- 主翻译角色只提取存在实质翻译疑问或全片一致性风险的疑难术语，不收集所有普通专名。
- 相同词形按义项归并；无法确信同义时保守拆分。术语语境默认前后各 10 个字幕段并允许调整。
- 代表语境集确定性选取，通常 3–5 个窗口、最多 10 个，优先覆盖首次出现、时间跨度、语义差异、歧义和冲突证据，不随机抽样。
- 每个候选批次保持“主翻译建议 → 高级校对裁决”的依赖顺序。高级校对先返回 `is_term`，为真时再返回接受、纠正并给出译法或首轮不确定。
- `is_term=false` 的自动排除项仍保留在审计记录中，不进入权威术语表。
- 首轮不确定时补充到最多 10 个代表语境进行第二轮。第二轮使用严格 schema，只允许接受或纠正并给出非空译法，不再包含 `is_term` 或不确定。
- 第二轮最多 3 次技术尝试。空响应、拒答、不可解析、非法状态、候选 ID 漏项／多项或纠正缺少译法均为机械响应错误；语义质量不由代码猜测。
- 第二轮耗尽后仅该术语回退为源词本身，任务继续，并在日志和翻译审计报告中标记校对回退及高风险。

## 人工术语确认工作台

- 交互式 GUI 任务选择人工确认时采用全尺寸流程页面并在该阶段主动暂停，不使用弹窗或可绕过的普通页签；没有候选时自动略过。CLI、批量和非交互 workflow 只使用自动确认。
- 页面采用主从布局：左侧为紧凑术语列表，右侧展示主模型建议、校对结论、当前最终译法、出现位置和代表语境，并支持上一项／下一项浏览。
- 页面不维护逐项待确认状态，也不要求逐项点击确认。每项初始采用高级校对形成的最终译法，用户可以直接继续，或改选主建议、自定义译法、合并候选、标记为不是术语／忽略。
- 用户标记为不是术语／忽略的候选不进入权威术语表，但保留在审计记录中。
- 不增加阻塞式输入校验；自定义空值也按用户明确选择原样保存，不自动恢复默认值。
- 合并时左侧列表进入多选；当前术语作为主项并保留其最终译法，其他候选的别名、语境和出现位置并入主项后不再独立显示。

## 项目术语表

- 自动生成 `【项目术语表】<文件名>.vcglossary.json`，保存 schema/version、语言对、规范化字幕指纹、义项、译文、别名、出现位置和选择来源。
- schema、语言对和字幕指纹精确匹配时，导入后恢复权威术语表并跳过术语提取、双模型建议和人工确认，但仍执行分层全文分析。
- 指纹不匹配但语言对兼容时只作为术语种子，重新扫描出现位置并进入确认流程，不得静默跳过。
- 第一版不实现 TBX；CSV 仅保留为未来人工交换格式，不承担精确恢复。
- 权威术语表完整持久化，但每个实际翻译块只注入块级相关术语。相关性先按出现字幕编号与当前主体／邻接语境求交，再以规范化词形和别名扫描补足。
- 块级相关性不使用向量检索，也不再次调用 LLM 判断；权威术语表不做生成式摘要。

## Prompt 与 token 规划

- 完整调用顺序固定为：系统硬约束 → 用户自定义角色 Prompt → 翻译上下文简报 → 当前调用相关权威术语 → 阶段指令 → 动态字幕。
- 系统硬约束与用户 Prompt 原文保留；上下文简报允许结构化递归压缩。用户 Prompt 不能覆盖输出可解析、字幕编号一一对应等硬约束。
- 工作上下文上限表示用户希望采用的单次总预算，而非模型技术极限，默认 64k tokens。预算包含固定 Prompt、动态输入、输出和推理空间。
- 工作上下文输入旁提供手动探查按钮，只尝试获取或验证模型技术上下文能力作为参考；不得自动覆盖用户预算，无法可靠取得时提示手动填写。
- 上下文超限时依次回退到 32k、16k并写警告，不覆盖保存值。
- 单模型与增强型 LLM 分别保存自己的 `batch_size`。增强型的值只计算正式翻译主体，是单次请求的最大输出字幕数；token 规划器可自动缩小。全文分析、术语处理和审计均按 token 自动分批，不提供各自批量大小设置。
- 每个翻译主体固定携带前后各最多 3 个源字幕段作为只读边界语境。主体和边界语境使用不同字段，模型输出只能包含主体 ID；输出边界 ID 时进入修正重试。
- 主体缩到 1 段仍超限时，依次进一步压缩上下文简报，并将边界语境从 3 降为 2、1、0；仍无法容纳才失败并报告各组成部分的 token 占用。
- 模型配置方案保存最大并发数。主翻译与高级校对绑定不同方案时使用独立并发限制；绑定同一方案时共享该方案限制。

## 接口适配与缓存

- 传输层与供应商方言分离。传输层支持 OpenAI-compatible、Anthropic Messages 和 Gemini；方言层支持通用、OpenAI、DeepSeek、Kimi、GLM、Qwen 等。
- 旧配置保持通用方言；供应商由用户手动选择，不根据 URL、模型名或响应自动识别。
- 缓存能力完全隐式，用户不配置缓存键、断点、TTL、预热或缓存资源。
- 不使用对话 fork 或 `previous_response_id` 规避输入费用。每个字幕块保持独立请求和并行能力。
- 稳定缓存前缀包含系统硬约束、用户 Prompt、压缩后的上下文简报及权威术语表版本；缓存断点位于稳定核心之后，块级术语与动态字幕位于断点之后。
- 适配层自动处理规范化前缀、预热、显式断点或缓存资源生命周期；不可用时回退普通请求，不阻断翻译。通用 OpenAI-compatible 模式同样保持稳定前缀并保留供应商扩展 usage。
- 供应商事实见 [Translation Prompt Caching Research](../research/translation-prompt-caching.md)。

## 翻译质量审计

- 高级校对分批覆盖全部源文和译文，检查语义忠实、漏译／增译／原文照抄、数字日期金额单位 URL 代码公式、否定情态、人物称谓与指代、权威术语与新疑难术语、跨段连贯、明显翻译腔／语法／语域，以及字幕 ID、占位符、标签和变量。
- 字幕 ID、非空输出、数字、显式符号、URL、代码、公式、占位符和标签先由本地确定性规则检查，再交高级校对复核并判断语义问题。
- 时间轴、CPS、段落合并拆分、断句、排版、间隙、尾部补偿和通用标点清理属于字幕后处理，不进入翻译审计。
- 高级校对在同一次响应中给出建议修正版，不交回主模型重译，也不增加一次高级校对复审。
- 自动修复只应用客观、高置信问题，并必须通过字幕 ID、非空输出、数字、占位符和允许修改范围等本地校验。自然度、语域、翻译腔和跨段表达偏好只报告。
- 校验失败时保留原译文并写入报告。“审计仅报告”页面严格只读，不提供逐项应用或提交修改。
- 内部返回单一结构化 `TranslationAuditReport`。GUI 独立任务、GUI workflow、CLI 和批量任务均在输出目录生成 `【翻译审计】<文件名>.md`；第一版不生成重复 JSON。

## 错误、重试与取消

- LLM 适配层重试超时、断线、429 和服务端 5xx 等瞬态传输错误，默认最多 4 次总尝试，使用带抖动指数退避并尊重 `Retry-After`。
- 增强翻译编排层对空响应、JSON／schema 错误和字幕 ID 缺失等机械错误附带反馈，默认最多 3 次总尝试，只重试当前调用或字幕块。
- 认证／权限失败、模型不存在、接口或协议配置错误、无效输入等永久错误不自动重试。
- 重试不暴露到第一版 UI；每次重试和最终耗尽写入日志。耗尽后抛出携带类别、阶段、可重试性、尝试次数和原始原因的结构化错误。
- 增强任务整体失败时保留已有中间产物，不自动降级为普通 LLM，也不把未经完整审计的字幕交给后处理。批量调度记录当前项失败后继续下一项。
- 取消时停止派发新请求并取消未开始批次；已经发出的请求允许自然结束但丢弃结果。任务标记已取消，不生成初版字幕、不进入后处理，已正式生成的项目术语表可以保留。
- 全软件统一错误码、跨模块策略、队列熔断、失败重跑和断点恢复延期到独立的任务错误处理项目。

## 设置与任务 UI

- 应用侧边栏保留一个「翻译设置」入口，内部用三段式分页切换“非 LLM 翻译｜LLM 翻译｜增强型 LLM 翻译”。分页仅为配置导航，不改变实际翻译方式。
- 任务入口使用三张并列卡片选择翻译方式。卡片显示用途、所需模型角色与配置状态，不显示“推荐”徽标；缺配置的卡片仍可选中，但禁用开始操作并提供设置入口。
- 非 LLM 面板选择 Bing、Google、DeepLX 等服务及其连接参数。单模型 LLM 面板显示主翻译方案、主 Prompt、反思翻译和批处理大小。增强面板显示两个角色方案、两份 Prompt、工作上下文、术语确认、审计处理和术语表导入，不显示反思翻译。
- 主翻译与高级校对各自只有一个持久化当前模型方案绑定。设置页和任务页若都提供选择控件，读写同一个值并即时同步，不存在默认值与本次覆盖值。
- 任务创建时解析角色绑定、模型配置和 Prompt 为不可变快照；运行中的设置变化只影响未来任务。
- 仅填写主翻译角色时，普通 LLM 可用、增强模式不可用；仅填写高级校对角色时，两种 LLM 模式均不可用。系统不自动复制新填写的角色配置。
- 人工术语确认和独立翻译审计结果使用全尺寸流程页面。交互式 GUI workflow 只有在用户主动选择人工术语确认时才在术语阶段暂停；任何 workflow 都不因末尾审计报告展示阻塞。CLI 和批量任务不进入交互页面。

## 迁移约束

- 新安装默认选择增强型 LLM 翻译。升级时旧 LLM 翻译迁移为增强型，并把原方案一次性绑定到两个角色；旧非 LLM 选择保持不变。
- 旧 `Subtitle/CustomPromptText` 迁移为主翻译 Prompt；高级校对 Prompt 使用新默认，不复制旧值。主翻译 Prompt 不再传给字幕优化器。
- 现有单一 OpenAI-compatible 全局 client 需要迁移到适配层。现有翻译器会立即并发全部块，本地翻译缓存键还遗漏用户 Prompt、上下文简报和权威术语等输入，实施时必须修正。

## Usage 统计

- 任务摘要和翻译审计报告按阶段、按角色汇总调用次数、输入 token、输出 token，以及供应商实际返回的缓存读取／写入 token。
- 接口未返回的 usage 字段明确标记不可用，不从字符数推算或伪造缓存命中。
- 第一版不估算货币金额；自定义接口的真实价格不可靠，未来如需计费由独立价格配置承担。
