# Subtitle Translation

字幕翻译负责在理解完整源字幕的基础上统一关键译名，并生成可交付给字幕后处理的初版字幕。

## Language

**翻译方式**:
用户在字幕翻译任务开始前选择的一条顶层工作流；当前分为非 LLM 翻译、单模型 LLM 翻译和增强型 LLM 上下文翻译，三者的配置作用域与运行行为相互独立。
_Avoid_: 翻译模型、术语确认模式、审计策略

**非 LLM 翻译**:
通过 Bing、Google、DeepLX 等非大语言模型翻译服务直接生成译文的翻译方式；它不进入全文分析、双角色术语确认或翻译质量审计。
_Avoid_: 普通模式、单模型 LLM 翻译、增强型翻译降级

**疑难术语候选**:
AI 从全文中识别出的、因含义、固定译名、音译方式、来源文本准确性或全片一致性而存在实质翻译疑问的词语；译法明确且无需统一决策的普通专名不属于候选。
_Avoid_: 特殊专有词汇、全部专名、未翻译词

**术语确认**:
在实际逐段翻译开始前，为每个疑难术语候选确定本次任务采用的权威译法；任务可采用自动确认或手动确认。
_Avoid_: 翻译后修正、表格浏览、模型投票

**自动术语确认**:
所有翻译任务默认采用的非阻塞术语确认方式，采用高级校对模型接受的主建议或给出的纠正译法，并在审计信息中明确标记该译法未经用户人工确认。
_Avoid_: 用户确认、静默默认、跳过术语处理

**人工术语确认**:
GUI 用户主动选择后采用的术语确认方式，任务进入交互步骤等待用户逐项选择主翻译建议、高级校对建议、自定义译法或忽略候选。
_Avoid_: 默认确认模式、CLI 确认、自动选择

**主翻译模型**:
一次字幕翻译任务中负责理解全文、识别疑难术语候选、首先提出术语译法并执行实际逐段翻译的模型角色；它使用主翻译提示词，不承担翻译质量审计或审计修正职责。
_Avoid_: 普通翻译器、高级校对模型、默认模型

**高级校对模型**:
在看到主翻译模型建议后进行有方向复核，并负责翻译质量审计及直接提出修正版的模型角色；它使用独立高级校对提示词，对术语建议作出接受、纠正或首轮不确定裁决，而不是独立生成并列答案。
_Avoid_: 并列翻译模型、主翻译模型、答案投票器

**校对裁决**:
高级校对模型在确认候选属于术语后，对主翻译术语建议作出的结构化结果：接受主建议、拒绝并给出纠正译法，或仅在首轮表示不确定；候选是否属于术语是独立判断，不占用这三个裁决状态。
_Avoid_: 双模型投票、独立候选、无结论评论

**校对回退**:
高级校对模型首轮返回不确定后，系统补充更多代表语境发起的第二轮最终裁决；第二轮协议只允许接受或纠正并给出非空译法，不再提供不确定或重新判断候选类别的状态。第二轮无法形成有效响应时，该术语保留源词不翻译，任务继续，并在审计报告和运行日志中标记高风险回退。
_Avoid_: 无限重试、第三轮判断、普通校对

**原文保留回退**:
疑难术语的第二轮校对无法形成有效裁决时，将该术语暂时映射为源词本身并继续翻译的降级结果；它不是用户确认译法，必须进入重点审计。
_Avoid_: 整段不翻译、确认不译、静默降级

**独立复核角色**:
由高级校对角色检查主翻译角色的翻译与修复结果，并使用独立提示词的质量约束；两个角色允许由用户选择相同的模型配置方案。
_Avoid_: 主翻译角色自检、共用提示词、强制不同模型

**增强型 LLM 上下文翻译**:
软件默认采用的翻译方式，由可用的主翻译模型和高级校对模型共同完成术语准备、再由主翻译模型执行逐段翻译；缺少高级校对模型时不得在运行中退化为主模型自审，Bing、Google、DeepLX 等普通翻译服务也不进入该流程。
_Avoid_: 双模型投票、所有翻译服务、反思翻译、机器翻译后重写

**单模型 LLM 翻译**:
用户在任务开始前显式选择、复用主翻译模型配置和主翻译提示词直接执行字幕翻译的独立翻译方式；它不提供增强型 LLM 上下文翻译的全文分析、术语确认和末尾独立复核。
_Avoid_: 上下文翻译降级、主模型自审、双模型翻译

**模型配置方案**:
由用户命名、可复用的一套模型连接与模型选择；主翻译模型和高级校对模型分别引用方案，任务选择器只展示方案名称。
_Avoid_: 任务内连接参数、模型角色、缓存配置

**工作上下文上限**:
用户为一个模型配置方案指定的单次调用总预算，包含固定提示词、动态输入、输出和推理空间；它允许主动低于模型技术上限以避免接近极限时的质量下降，默认值为 64k tokens。
_Avoid_: 模型最大上下文、全文分析窗口、输出长度

**主翻译提示词**:
专门约束主翻译角色如何理解、翻译和修复字幕的可配置任务指令；任务开始时冻结快照，不与模型配置方案绑定。
_Avoid_: 高级校对提示词、模型连接配置、运行中共享文本

**高级校对提示词**:
专门约束高级校对角色如何裁决主翻译术语建议、执行翻译质量审计和给出修正版的可配置任务指令；任务开始时冻结快照，不与主翻译提示词共用。
_Avoid_: 主翻译提示词、模型连接配置、主模型自检指令

**完整翻译提示词**:
一次模型调用所采用的任务级指令组合，依次包含系统硬约束、对应角色的用户自定义提示词、翻译上下文简报、当前调用所需的权威术语和当前阶段指令；动态字幕内容位于其后。
_Avoid_: 主提示词、文稿参考、单段字幕请求

**权威术语表**:
保存一次任务全部用户确认术语及其义项、译法、来源和确认方式的完整任务数据；它不能被生成式摘要替代或静默删除。
_Avoid_: 提示词片段、术语摘要、全局长期记忆

**项目术语表文件**:
术语确认完成后自动生成、可由用户在以后任务开始前显式导入的版本化 JSON 翻译资产；它保存语言对、源字幕指纹和权威术语表，但不代表任务可从中断处恢复。
_Avoid_: 任务恢复文件、翻译缓存、通用术语库

**块级术语集**:
从权威术语表中为当前字幕块确定性选出的精确术语子集；相关性以候选记录的出现字幕段编号为主，并通过规范化词形和别名扫描补足。
_Avoid_: 完整术语表、语义摘要、随机术语抽样

**翻译主体**:
一次实际翻译调用中必须生成译文的连续字幕段集合；模型输出只能包含这些字幕段的编号。
_Avoid_: 翻译批次全部输入、边界语境段、上下文窗口

**边界语境段**:
翻译主体之前和之后各最多 3 个、只供理解而不得生成译文的源字幕段；它们与翻译主体在请求结构和输出校验中明确分离。
_Avoid_: 待翻译字幕、重叠翻译段、术语语境窗口

**分层全文分析**:
覆盖全部源字幕、但允许先分窗口分析再汇总的翻译准备过程；“全文”描述覆盖范围，不表示必须把全部字幕放进一次模型请求。
_Avoid_: 单次全文请求、随机抽样、逐段翻译

**全文分析窗口**:
分层全文分析中按 token 预算形成、在字幕段边界切分并与相邻窗口保留少量重叠的一组源字幕；窗口输出携带原始字幕编号，以便汇总时消除重叠。
_Avoid_: 固定字幕数量、术语语境窗口、无编号文本块

**翻译上下文简报**:
分层全文分析汇总形成的任务级上下文，概括内容大纲、背景、主题及影响译法的表达特征，供术语判断和实际逐段翻译共同使用。
_Avoid_: 主提示词、完整原文、局部摘要

**术语语境窗口**:
以疑难术语候选所在字幕段为锚点、向前和向后各取若干字幕段形成的判断依据；默认范围为前后各 10 段，并允许用户调整。
_Avoid_: 前后十句话、完整原文、翻译批次

**术语义项**:
同一词形在特定语境中表达的一个具体含义或指代对象；相同词形的不同义项必须作为不同的疑难术语候选分别确认。
_Avoid_: 字符串去重项、同义词、全部出现位置

**保守义项拆分**:
AI 无法确信多个出现位置表达同一术语义项时，将其保留为多个候选而不自动合并；用户可在术语确认时显式合并。
_Avoid_: 猜测合并、字符串去重、重复候选错误

**代表语境集**:
从一个术语义项的全部出现位置中确定性选出的语境窗口集合，通常包含 3 至 5 个、最多 10 个，并优先覆盖首次出现、时间跨度、语义差异、歧义和冲突证据。
_Avoid_: 随机抽样、全部语境、固定三个窗口

**术语确认步骤**:
GUI 人工术语确认模式在翻译准备与正式逐段翻译之间进入的页面内阻塞阶段；没有疑难术语候选时自动略过，不能作为普通分页签绕过。
_Avoid_: 术语确认页签、弹窗、可选浏览页

**翻译质量审计**:
高级校对模型在正式翻译完成后、交付初版字幕前分批覆盖全部源文与译文，对语义忠实、漏译增译、关键事实、人物称谓、术语、跨段连贯、目标语质量和格式保真进行的末次检查；它不属于字幕后处理，也不进入下游模块。
_Avoid_: 术语审计、字幕后处理审计、最终视频审计

**翻译审计报告**:
翻译质量审计形成的任务产物，记录发现的问题、建议译文、自动修复结果、校验失败和术语回退；独立任务可交互展示，流水线任务只保存而不阻塞后续阶段。
_Avoid_: 字幕后处理 QA 报告、运行日志、人工确认关卡

**独立字幕翻译任务**:
用户直接运行字幕翻译模块并以初版字幕和翻译审计报告为最终结果的 GUI 任务；完成后可以展示交互式审计结果页面。
_Avoid_: 完整 workflow、批量任务、字幕后处理阶段

**流水线翻译阶段**:
完整 workflow 中自动产生初版字幕并交给字幕后处理的字幕翻译调用；审计报告是随任务保存的产物，不是需要用户查看或确认的阻塞步骤。
_Avoid_: 独立字幕翻译任务、人工审计关卡、字幕后处理

**审计仅报告**:
翻译质量审计只形成可定位问题项而不修改字幕的处理策略，用户可据此手动编辑；它是 GUI 单任务的默认策略。
_Avoid_: 自动修复、静默修改、仅写日志

**自动修复客观问题**:
高级校对模型在翻译质量审计中直接给出客观、高置信问题字幕段的修正版，本地确定性校验通过后才应用；主观质量建议只报告，校验失败的修正版不得应用并保留在报告中。它是 CLI 和批量任务的默认策略。
_Avoid_: 交回主模型重译、整片重译、无校验替换
