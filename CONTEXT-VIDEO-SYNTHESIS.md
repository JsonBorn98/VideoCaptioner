# Video Synthesis Encoding

字幕视频合成负责把成型字幕烧录进画面（硬字幕）或作为独立轨道内嵌（软字幕），并按用户指定的编码格式与参数产出面向观众交付的最终媒体文件；它是完整 workflow 中字幕内容处理之后的最后一个阶段。

## Language

**视频合成**：
将成型字幕烧录或内嵌进视频、并按选定编码设置产出最终媒体文件的功能页面与 workflow 阶段。
_Avoid_: 字幕合并、合成视频、字幕视频合成阶段

**编码格式**：
视频压缩标准本身——AVC（H.264）、HEVC（H.265）、AV1；只描述"用哪种标准压"，不含由谁来压。
_Avoid_: 编码器、视频格式、容器

**编码器**：
实现某种编码格式的具体 ffmpeg 编码器，绑定运行后端（libx264、h264_nvenc、hevc_qsv…）；用户在单一分组列表中直接选择它。
_Avoid_: 编码格式、编解码、解码器

**硬件编码器**：
运行在 GPU/专用编码单元上的编码器（NVENC / QSV / AMF），是"利用显卡加速合成"的真正来源，区别于仅作用于解码侧的硬件加速。
_Avoid_: 硬件加速、-hwaccel、显卡解码

**直通**：
一种视频编码器选择项，直接流复制源视频而不重编码；仅在不做硬烧录时可用（硬烧录必须重编码）。
_Avoid_: 无损、软字幕、复制音轨

**编码器可用性探测**：
在固定策划的编码器清单之上，通过解析 ffmpeg 核心并对硬件编码器做真实初始化，判定某编码器在当前核心与硬件上能否实际运行，用于置灰不可用项。
_Avoid_: 全量动态列举、编译期开关、编码时才报错

**固定品质模式**：
以一个恒定质量数值控制压缩、码率随画面复杂度浮动的编码方式；该数值按所选编码器映射到其原生参数与范围（x264/x265/nvenc 等 0–51、AV1 0–63，越小越好），而非统一假刻度。
_Avoid_: RF 统一刻度、CRF 专属、码率模式

**平均码率模式**：
以目标码率（kbps）为主控的编码方式；用户手填模板码率。
_Avoid_: 固定品质、CBR、质量档位

**多阶段编码**：
平均码率模式下先分析后编码的两遍编码，仅对 CPU 编码器（x264/x265/SVT/aom）可用；硬件编码器走单遍。
_Avoid_: 硬件 multipass、单遍 ABR、二次转码

**首遍加速**：
多阶段编码首遍采用的加速分析设置，默认开启，仅在 CPU 多阶段编码时出现。
_Avoid_: turbo 关闭、正式遍、快速预设

**分辨率高度模板**：
分辨率预设与自定义都以目标高度为模板，宽度按源宽高比自适应并取偶数；目标高度只作上限，源不高于目标时保持源分辨率不放大。
_Avoid_: 目标宽度、任意 W×H、强制缩放

**色彩透传**：
重编码时尽力让输出与源在色彩范围、原色、传递特性、矩阵与像素位深上保持一致（含 HDR 元数据尽力透传），使成片"看起来不变"。
_Avoid_: 只保颜色范围、强制 8-bit、默认色彩

**音频直通**：
音频编码器选择项之一，直接流复制源音轨而不重编码，是音频默认行为；与目标容器不兼容时提示并回退重编码为 AAC。
_Avoid_: 重采样、复制视频、无损音频

**起始归零**：
封装阶段将各轨道时间戳统一平移使输出从 0 开始的良性规整（`-avoid_negative_ts make_zero`），不改变音画相对同步，也不影响已烧录字幕落帧。
_Avoid_: 音画重对齐、修复延迟、丢帧对齐

**圆角背景**：
一种社交/短视频风格的字幕外观，用 Pillow 将每条字幕渲染为带圆角矩形底框的透明 PNG 再 overlay 叠加到视频，区别于 libass 的 ASS 样式硬烧；是次要可选渲染模式，仅其最终编码接入集中命令构建器。
_Avoid_: ASS 样式、软字幕、字幕特效

**集中命令构建器**：
由结构化编码设置生成 ffmpeg argv 的单一入口，被软/copy、ASS 硬烧、圆角最终编码共用，是命令预览与实际执行的共同来源。
_Avoid_: 各路自拼命令、字符串模板、三处硬编码

**命令预览**：
页面上与控件实时双向同步、可编辑并带校验的完整 ffmpeg 命令展示；采用白名单最佳努力反解析，未识别部分归入自定义参数。
_Avoid_: 只读日志、完整反解析、脚本框

**自定义参数**：
附加在构建命令末段的用户自填 ffmpeg 参数；CLI 另有整条命令逐字执行（raw）与命令打印（print）构成的透传闭环。
_Avoid_: x264-params 专属、shell 命令、覆盖字幕滤镜

**视频合成前缀**：
输出文件固定前缀「【视频合成】」，与字幕阶段前缀风格一致；后缀按 `_{高度}p_{编码器}_{编码格式}_{质量}` 编码，直通则为 `_{高度}p_copy`。
_Avoid_: 【卡卡】、随机命名、宽度后缀

**编码器选项目录**：
每个编码器可选 preset/微调/快速解码/配置/级别与质量参数范围的策划数据表，是 UI 选项渲染与参数映射的权威来源。
_Avoid_: 动态解析 -h encoder、硬编码默认、全量笛卡尔积
